Implementation Plan: QA Automation

Thiết lập hệ thống kiểm thử tự động toàn diện từ Unit, Integration đến Performance.

1. Overview (Tổng quan)

Module QA Automation giải quyết vấn đề thiếu hụt test case bằng cách tận dụng 5 Skills chuyên dụng để tự động hóa việc sinh kịch bản, kiểm thử tích hợp API và đo lường hiệu năng. Mục tiêu là đạt coverage > 90% và đảm bảo an toàn cho hàm validate_sql.

2. Các bước triển khai chi tiết

Bước 1: Thiết lập Integration Testing Framework

Mục tiêu: Xây dựng bộ test tích hợp cho toàn bộ Endpoints microservices.

  • Tác vụ 1: Sử dụng skill api-integration-expert viết Pytest cho Backend.
  • Tác vụ 2: Viết Unit tests chuyên sâu cho hàm validate_sql.
"Dựa trên cấu trúc thư mục services/, hãy sử dụng skill api-integration-expert để viết bộ Integration Tests hoàn chỉnh cho 3 dịch vụ. Đặc biệt, hãy viết riêng một file test_security.py để kiểm thử hàm validate_sql với 20+ patterns injection phổ biến. Kết quả: Bộ test Pytest đạt coverage > 80%."

Bước 2: Tự động hóa kịch bản & Hiệu năng

Mục tiêu: Sinh test case tự động và chạy Load Test k6.

  • Tác vụ 1: Sử dụng skill Automated Test Case Generation sinh kịch bản.
  • Tác vụ 2: Chạy k6 load test cho endpoint /ask và dashboard refresh.
"Sử dụng skill Automated Test Case Generation để sinh ra 50 kịch bản kiểm thử cho toàn hệ thống. Sau đó, hãy dùng skill k6-performance để chạy Load Test với 100 concurrent users cho endpoint /ask. Kết quả: File kịch bản .csv và kết quả k6 report."

Bước 3: Tổng hợp báo cáo kiểm thử (Test Reporting)

Mục tiêu: Xuất báo cáo chất lượng tổng hợp định kỳ.

  • Tác vụ 1: Sử dụng skill Test Report Writing tổng hợp kết quả.
  • Tác vụ 2: Xây dựng script run_all_tests.sh để tự động hóa quy trình.
"Sử dụng skill Test Report Writing để tổng hợp toàn bộ kết quả kiểm thử từ Bước 1 và Bước 2 vào một file báo cáo Markdown chuyên nghiệp. Đảm bảo nêu rõ tỷ lệ Pass/Fail và các khuyến nghị về hiệu năng. Kết quả: File QA_REPORT_FINAL.md."
